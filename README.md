# Bill Splitter — Разделение счетов

## О проекте

**Bill Splitter** — веб-приложение для справедливого разделения общих расходов в компаниях, поездках или мероприятиях. Сервис автоматически рассчитывает, кто кому и сколько должен, поддерживает два алгоритма распределения и сохраняет историю счетов.

### Основные возможности

- Создание и управление несколькими счетами
- Добавление участников, товаров и платежей
- Два метода расчёта долгов:
  - **Равное разделение** — общая сумма делится поровну
  - **Пропорциональное** — каждый платит только за то, что потреблял
- Автоматическая оптимизация транзакций (минимальное количество переводов)
- Сохранение всех данных в JSON-файлы
- Веб-интерфейс с адаптивным дизайном

### Технологический стек

## Быстрый старт

### Локальный запуск

```bash
# 1. Клонирование репозитория
git clone https://github.com/waqque/splitbill
cd splitbill

# 2. Создание структуры папок
make init

# 3. Установка зависимостей
pip install -r requirements.txt

# 4. Запуск приложения
python -m src.web
# или
make run
```

После запуска откройте браузер: http://localhost:8000

```bash
# Сборка образа
make build

# Запуск контейнера
make up

# Просмотр логов
make logs

make down
```

###  Логика работы приложения

#### 1. Управление состоянием
- Приложение хранит текущий активный счёт в глобальной переменной `current_bill`
- Каждое действие (добавление пользователя/товара/платежа) автоматически сохраняет счёт в `data/{bill_id}.json`
- Пользователь может переключаться между счетами или создавать новые

#### 2. Алгоритмы расчёта долгов
**Равное разделение** (`equal`)
```python
total = sum(item.price * item.quantity for item in items)
per_person = total / len(users)
balance[user] = paid[user] - per_person
```
Когда использовать: все участники потребляли примерно поровну.

**Пропорциональное разделение** (`proportional`)
```python
for item in items:
    per_person = item.price / len(item.consumers)
    for consumer in item.consumers:
        consumed[consumer] += per_person
balance[user] = paid[user] - consumed[user]
```
Когда использовать: разные участники потребляли разные товары.

#### 3. Оптимизация транзакций
Функция _simplify_debts() минимизирует количество переводов:
- Сортирует должников (отрицательный баланс) и кредиторов (положительный)
- Погашает наибольший долг наибольшим требованием
- Исключает суммы менее 0.01 (погрешность округления)

#### 4. Хранение данных
- Каждый счёт — отдельный JSON-файл в `data/`
- Имя файла: `{bill_id}.json`
- Сериализация: `datetime `→ ISO-строка, `Decimal` → float

## Формат JSON-файлов
```json
{
  "id": "uuid",
  "name": "Ужин в ресторане",
  "users": [{"id": "...", "name": "Анна"}],
  "items": [{"name": "Пицца", "price": 500.0, "consumers": ["..."], "quantity": 1}],
  "payments": [{"user_id": "...", "amount": 1000.0}],
  "settlements": []
}
```

### Для `calculator.py`:

## Алгоритм упрощения долгов
1. Сортировка должников (от большего долга к меньшему)
2. Сортировка кредиторов (от большего требования к меньшему)
3. Погашение наибольшего долга наибольшим требованием
4. Исключение сумм < 0.01 (погрешность округления)

## Makefile команды

`make init` - Создать структуру папок (`src/`, `templates/`, `static/`, `data/`, `tests/`)

`make run` - Запустить Uvicorn сервер на порту 8000

`make build` - Собрать Docker-образ

`make up` - Запустить контейнер в фоновом режиме

`make down` - Остановить контейнер

`make logs` - Показать логи контейнера

`make clean` - Удалить кэш (`__pycache__/`) и все JSON-файлы из `data/`
